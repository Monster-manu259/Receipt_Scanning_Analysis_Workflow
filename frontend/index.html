AI Structuring
Groq LLM parses OCR output into clean fuel-bill JSON fields including receipt no, nozzle no, product, density, rate, volume, amount, and totals.

Your dashboard computes spending trends, category breakdowns, and month-over-month comparisons dynamically from your uploaded data.

Track transactions by nozzle number and pump metadata for better station-level transaction history.

Project Overview

Fuel Bill Scanning
Analysis Workflow

A production-grade full-stack pipeline that accepts any fuel-bill image or PDF, runs it through OCR, passes raw text to an LLM for structured extraction, persists results to a database, and surfaces real-time analytics — all with a clean API surface.

Processing Pipeline
01
Ingestion
Accept multipart uploads (JPG, JPEG, PNG, PDF) and normalize image/PDF pages for OCR extraction.
02
OCR Extraction
Tesseract OCR extracts full receipt text after preprocessing with PIL, contrast enhancement, and sharpening.
03
AI Structuring
Groq-hosted Llama parses OCR text into structured fuel-bill JSON with receipt no, nozzle no, product, density, rate, volume, and amount.
04
Persistence
Structured payload is stored in PostgreSQL JSONB for reliable fuel transaction history and fast analytics.
05
Analytics
Aggregation pipeline computes monthly fuel spend, product mix, average volume, and trend signals in real time.
06
API Surface
FastAPI exposes REST endpoints: POST /upload, GET /receipts, DELETE /receipts/{id}, GET /insights — all OpenAPI documented.
